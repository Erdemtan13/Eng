# Eng

**Eng is a simple, English-based programming language designed to make programming easier to understand.**

Instead of requiring complicated syntax, Eng lets you write code using plain, readable English.

```eng
When started:
    This is name
    Ask the user "What is your name?" and put the answer in name

    Say "Hello, " and name
```

## ⚠️ Pre-Alpha

Eng is currently in the **pre-alpha stage**.

This means that the language is still very early in development, and many important features may not exist yet.

If you think Eng is missing a function that a programming language needs, please report it in the **Issues** tab.

Please especially report:

* Missing essential programming features
* Functions that would be useful in many programs
* Bugs
* Commands that do not work as expected
* Suggestions for improving the language

Your feedback can help shape the future of Eng.

## What can Eng currently do?

Eng currently supports features such as:

* Variables
* Numbers
* Strings
* User input
* Yes/no input
* Number input
* Mathematical operations
* Addition
* Subtraction
* Multiplication
* Division
* Powers
* Square roots
* String length
* Conditional statements
* Comparisons
* Repeat loops
* Boolean values
* `Stop`
* `Stop then continue`

Example:

```eng
When started:
    This is age
    Ask the user for a number and put the answer in age

    If age is more than 10:
        Say "You are older than 10."
```

## Running Eng

After installing the Eng interpreter, run an Eng program with:

```bash
eng program.eng
```

For example:

```bash
eng fibonacci.eng
```

## Building Eng

Eng is currently distributed as platform-specific executables.

Supported platforms include:

* macOS Intel
* macOS Apple Silicon
* Windows x64
* Linux x64

## Contributing

Eng is still in its early stages, and contributions are welcome.

If you find something missing, broken, or confusing, please open an issue.

Before opening an issue, please check whether a similar issue already exists.

## The Future of Eng

Eng is intended to eventually become a complete programming language with an ecosystem of libraries.

Possible future libraries include:

* `engame` — Game development
* `numbereng` — Numerical computing
* `neuralnengwork` — Neural networks
* `webeng` — Web development
* `soundeng` — Audio
* `graphiceng` — Graphics

The goal is to make programming feel more natural while still allowing people to build real software.
